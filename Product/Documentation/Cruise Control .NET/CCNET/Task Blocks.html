CruiseControl.NET : Task Blocks
This page last changed on Sep 25, 2005 by mroberts.

Types of Task Block are:

  • Build Publisher
  • Email Publisher
  • Executable Task
  • File Merge Task
  • ForceBuildPublisher
  • Modification Writer Task
  • MsBuild Task
  • NAnt Task
  • Null Task
  • NUnit Task
  • Visual Studio Task
  • Xml Log Publisher

The <tasks> section is for build tasks, for example a Nant Task or MsBuild Task. If any of these build tasks fail, no further build tasks will be executed. Task that publish the results of the build should be placed in the <publishers> section.

The <publishers> section is run after the build completes (whether it passes or fails). This is where you aggregate and publish the build results. It typically starts with a <merge> task to collect any output from nant, nunit or other tools, so that the results can be shown on the dashboard.

Note: Publishers and Tasks were different objects in earlier version of ccnet. Now they are interchangeable, and can appear either in the <tasks> section or the <publishers> section of the ccnet.config file depending on whether they should be run in or after the build.
