CruiseControl.NET : Trigger Blocks
This page last changed on Aug 15, 2005 by orogers.

Trigger blocks allow you to specify when CruiseControl.NET will start a new integration cycle.

Typically, you'll want to use an Interval Trigger. Also useful is the Schedule Trigger for implementing daily builds. Use the Filter Trigger to prevent builds from occurring at certain times (such as when your source control repository is undergoing backup). If you want to set up a project that will only respond to force build requests, you should specify an empty trigger section like this: <triggers/>.

Types of Integration Trigger Block are:

  • Filter Trigger
  • Interval Trigger
  • Project Trigger
  • Schedule Trigger
  • Url Trigger
